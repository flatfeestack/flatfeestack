What are our fees?

Since we love open source software and this being a side-project, we charge 0%. That means building this site, hosting fees, accounting, etc. is donated by us. We would like to distribute the full amount you payed to the projects, however, there are service fees ~3% (e.g., credit card fees) that is deducted from your $5.

What about Open Collective, Patreon, or Github Sponsors?

Donation services support single project or person based contributions: e.g., support project X with $50 per year, or support person Y with $50 per year. The problem is that the more projects or people you support, the more you pay.

From our personal experience, when we come across an interesting project, we often end up not supporting, as we would have to support at least 50 other interesting projects. Paying for every single project or person does not work for us and we would rather have a single flat fee to support those 50 projects.

The solution is flat fee contribution. Each project receives your donation equally divided. Simply donate and don't worry about how many projects you can support, its always $5.

It is a similar concept to Flattr for open source projects, where git commits decide how much each contributor earns.

The flatfeestack Process in 4 Steps

1. Register Project

Any backer, earner or sponsor can register a project with a public git repository. At the moment, only git is supported.

2. Support Project

Any backer or earner can support a project that was registered in step 1. Simply click an a badge or search the project in our database.

3. We Monitor Contributions

Flatfeestack monitors the public repository and stores developer contributions. A link between flatfeestack users and the git user/email has to be done.

4. Payout

Payout is done in a 3 month intervals. Within these 3 months the payout is proportional to your contribution.

FAQ

What happens if a developer does not claim his earnings?

If a backer supports 16 projects and in 1 project no developer claims his earnings, the backer will give the money to the other 15 projects. If within a project with 4 developers, one developer does not claims his earnings, the others 3 splits the donation.

When does the payout to a developer happen?

We have 3 month periods and if after 3 months a developer does not setup his account, the developer does not get his share.

Why is the payout in Cryptocurrency?

To keep the fees as low as possible and due to much easier integration we use Ethereum. In future we plan to support other crypto currencies.

How can a developer claim that he owns a username?

To claim payment, in the root repo, there needs to be a directory .flatfeestack/. Each developer commits/pushes a property file with the name of his/her flatfeestack UUID. For example tbocek-366bc810.prop. For GitHub we also provide a service where any GitHub user can create a pull request that gets accepted automatically: https://github.com/flatfeestack/users. The property file has the following content: uuid = 66cbc110-9db2-4ca3-9082-49e2a0813af7 /* mandatory */

How do we measure contributions?

We go a git log and add the number of lines changes: git log --after="2019-01-01" --before="2019-04-01" --pretty=%aE --numstat
